h1. Overview

h4. Below is a list of new features, improvements, fixed issues, and known issues in Cloudify.
We track all the issues in our public "JIRA":https://cloudifysource.atlassian.net.

"Cloudify 2.2.0 GA":#v220ga  - published on Oct 24, 2012
This version contains the following key improvements:

* "Tail logs on CLI":../recipe/command_line_reference#tail - You can tail any Cloudify log from the Cloudify shell.
* "Cloud Driver Parametrization":../clouddrivers/cloud_driver#gstr - Users can use Groovy GString expressions in the cloud driver DSL.
* View recipes in the web UI - We now eable users to browse the deployed recipes from the Web UI.
* "Run as a sudoer":../setup/configuring_openstack - Users can install applications as sudoers on linux machines (works on BYON as well).
* "OS variables in a template":../clouddrivers/cloud_driver#sos - You can set or override OS environment variables in Cloud Driver template.
* "Rest API":../rest_api/rest_api_attributes -The REST API is now officially public.
* "Azure IaaS API":../setup/configuring_azure - Enables users to bootstrap and install an application on Windows Azure using the IaaS API.


<hr/>

"Cloudify 2.2.0 RC":#v220rc  - published on Oct 4, 2012
This version contains the following key improvements:
*  Azure IaaS -  Azure IaaS is now supported.
* "BYON as a sudoer":../bootstrapping/bootstrapping_byon -  Users can now install recipes on BYON as sudoers.

<hr/>

"Cloudify 2.2.0 M5":#v220m5  - published on Sep 23, 2012
This version contains the following key improvements:
* "Local cloud footprint":../bootstrapping/bootstrapping_localcloud - The memory footprint of local cloud has been dramatically reduced. Now all the local cloud's components are packed into 2 JVMs. 
* "Ubuntu Support in EC2":../setup/configuring_ec2 -  A new Ubuntu template (SMALL_UBUNTU) has been added to the default EC2 cloud driver.

<hr/>

"Cloudify 2.2.0 M4":#v220m4  - published on Sep 9, 2012
This version contains the following key improvements:
* "Rest API":../rest_api/rest_api_attributes - Cloudify's context attributes are now exposed over a REST API.
* "Azure IaaS API":../setup/configuring_azure - Users are now able to bootstrap and install an application on Windows Azure using the IaaS API.

<hr/>

"Cloudify 2.2.0 M2":#v220m2  - published on Aug 6, 2012
This version contains the following key improvements:
* "Attributes Per Template":../setup/configuring_ec2 -  Many attributes are now per template in the cloud-driver file. For example: localDirectory, remoteDirectory and keyFile.
* "Run as a sudoer":../setup/configuring_openstack - Users are now able to bootstrap and install an application as a sudoer on linux machines, just by setting the privileged attribute to true in the cloud-driver file.

<hr/>

"Cloudify 2.1.1 GA":#v211ga  - published on July 8, 2012
This version contains the following key improvements:
* "Process Locator":../developing/lifecycle_events#locator -  Enables recipes to specify which processes will be monitored for process metrics and process failures.
Utility functions are provided to locate processes by names, as well as by a query language "(newwindow)(SIGAR PTQL)":http://support.hyperic.com/display/SIGAR/PTQL.
* We've added lots of WEB UI enhancements.


<hr/>

"Cloudify 2.1.1 M2":#v211m2  - published on June 7, 2012
This milestone contains the following key improvements:
* "Process Locator":../developing/lifecycle_events#locator -  Enables recipes to specify which processes will be monitored for process metrics and process failures.
Utility functions are provided to locate processes by names, as well as by a query language "(newwindow)(SIGAR PTQL)":http://support.hyperic.com/display/SIGAR/PTQL.
* We've added lots of WEB UI enhancements.


<hr/>

"Cloudify 2.1.0 GA":#ga21  - published on May 2, 2012
This milestone contains the following key improvements:
* "Scaling rules":../developing/scaling_rules - Enables services to define scaling rules for scaling out or shrinking in.
* The Cloudify Interactive Shell now offers an improved user experience: Cloud bootstrapping, Application installation and uninstallation, as well as cloud teardown. These improvements facilitate progress tracking and simplify troubleshooting and recipe debugging.
