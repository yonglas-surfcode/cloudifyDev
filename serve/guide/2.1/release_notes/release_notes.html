Overview

Below is a list of new features, improvements, fixed issues, and known issues in Cloudify.
We track all the issues in our public JIRA.

Cloudify 2.2.0 RC – published on Oct 4, 2012
This version contains the following key improvements:

  • Azure IaaS – Azure IaaS is now supported.
  • BYON as a sudoer – Users can now install recipes on BYON as sudoers.

Cloudify 2.2.0 M5 – published on Sep 23, 2012
This version contains the following key improvements:

  • Local cloud footprint – The memory footprint of local cloud has been dramatically reduced. Now all the local cloud’s components are packed into 2 JVMs.
  • Ubuntu Support in EC2 – A new Ubuntu template (SMALL_UBUNTU) has been added to the default EC2 cloud driver.

Cloudify 2.2.0 M4 – published on Sep 9, 2012
This version contains the following key improvements:

  • Rest API – Cloudify’s context attributes are now exposed over a REST API.
  • Azure IaaS API – Users are now able to bootstrap and install an application on Windows Azure using the IaaS API.

Cloudify 2.2.0 M2 – published on Aug 6, 2012
This version contains the following key improvements:

  • Attributes Per Template – Many attributes are now per template in the cloud-driver file. For example: localDirectory, remoteDirectory and keyFile.
  • Run as a sudoer – Users are now able to bootstrap and install an application as a sudoer on linux machines, just by setting the privileged attribute to true in the cloud-driver file.

Cloudify 2.1.1 GA – published on July 8, 2012
This version contains the following key improvements:

  • Process Locator – Enables recipes to specify which processes will be monitored for process metrics and process failures.
    Utility functions are provided to locate processes by names, as well as by a query language (SIGAR PTQL).
  • We’ve added lots of WEB UI enhancements.

Cloudify 2.1.1 M2 – published on June 7, 2012
This milestone contains the following key improvements:

  • Process Locator – Enables recipes to specify which processes will be monitored for process metrics and process failures.
    Utility functions are provided to locate processes by names, as well as by a query language (SIGAR PTQL).
  • We’ve added lots of WEB UI enhancements.

Cloudify 2.1.0 GA – published on May 2, 2012
This milestone contains the following key improvements:

  • Scaling rules – Enables services to define scaling rules for scaling out or shrinking in.
  • The Cloudify Interactive Shell now offers an improved user experience: Cloud bootstrapping, Application installation and uninstallation, as well as cloud teardown. These improvements facilitate progress tracking and simplify troubleshooting and recipe debugging.
