This topic provides instructions for configuring Windows Azure for Cloudify.

Prerequisites

  • Windows 7, Windows Server 2008, or Windows Vista SP1 with the latest service packs installed
  • Windows Azure SDK Azure SDK v1.6 64-bit [x64] or Azure SDK v1.6 32-bit [x86]
  • A Windows Azure account
  • Silverlight-enabled web browser (required by Azure Management Portal)
  • Java Development Kit 1.6
  • Cloudify shell (for information about installing the Cloudify shell, see Installing the Cloudify Shell)

For information about Windows Azure terminology, see Windows Azure Glossary.

Understanding Windows Azure Security Credentials

Understanding Windows Azure security credentials is necessary in order to successfully manage a Java application running on Azure.

Windows Live Credentials
Credentials consist of the username and password used to log into the Azure Management Portal. Because Windows Live does not support two-factor authentication, make sure you use a strong password.

Remote Desktop Personal inFormation eXchange (RDP PFX) Certificate
The RDP PFX certificate contains both a private and public key, and is attached to an Azure deployment. After the PFX certificate is created on your local machine, you must upload it to a specific deployment. The PFX file is protected by a password when uploading the certificate to Azure. The certificate in the PFX file is used to encrypt and decrypt the Windows password that you use to log in to your Azure VMs (role instances) via Remote Desktop.

Do not confuse the password that protects the PFX file, with the Windows password that is encrypted using the certificate saved in the PFX file and that you use to log in to your Azure VMs.

PFX files contain both the private and the public keys, whilst CERT files contain only the public key.

Management API (REST) X509 Certificates
Management certificates are used to authenticate the REST API client (for example, when running a scaling-out command). For the authentication to occur, the public key (CERT file) must be uploaded to the “Management Certificates” section of your Azure Management Portal, and the private key (PFX file) must be installed in the client machine’s local certificate store.

Configuring Windows Azure

This section provides step-by-step instructions for the following configuration tasks:

  • Configuring the Azure properties file
  • Configuring Remote Desktop certificates
  • Configuring Management certificates
  • Updating the Blob Storage details

Configuring the Azure properties file

  1. Login to your Azure Management Portal.

    If the Isolated Storage Quota dialog box opens, click Change Quota to allow Windows Azure’s Silverlight application to use your local disk for temporary storage.

  2. On the views pane, click Hosted Services, Storage Accounts & CDN.
  3. On the navigation pane, click Hosted Services.
  4. Select the Azure Subscription row.
  5. From the Properties pane, copy the Subscription ID value.
  6. Edit the azure.properties file located in <cloudify root>\tools\cli\plugins\azure\, find the subscriptionId property, and then paste the copied Subscription ID value.

    In the azure.properties file included in the Cloudify distribution file, the cspackFolder property defines the path to the Windows Azure SDK. If the path is not correct for your installation, then specify the correct path.

Configuring Remote Desktop certificates (Optional)

This section describes how to create the certificates for remote access to your VMs. For this example, we will use the password 123456 to encrypt the PFX files.

  1. Create a protected Azure management (private and public) certificate—At the Windows command prompt, navigating to the Azure plugins folder, and then create the certificate, as per the following example:
  2. Choose a Windows password to use when logging in to your Azure VMs (role instances), and then encrypt it using the certificate—At the Windows command prompt, create the password and save it in password.txt, and then Notepad to copy the saved password.

    Choose a password with at least 10 characters, containing capital letters, lower case letters, and numbers.


  3. Edit the azure.properties file, find the rdpLoginEncryptedPassword property, and then paste the copied encrypted password value. Find the rdpLoginUsername property, and then specify the value as “Administrator”. This is the Windows Operating System user name that is used to log in with Remote Desktop.

Configuring Management (API) certificates (Optional)

This section describes how to configure the management certificates that are required to change the number of role instances (scale-out) from the Cloudify shell.

  1. Create the management certificate using the following steps:
    1. Create the certificate—At the Windows command prompt, type:
      encutil -create -alias azure_api -pwd 123456 -pfx azure_api.pfx -cert azure_api.cer
    2. Save the certificate thumbprint—At the Windows command prompt, type:
      encutil -thumbprint -cert azure_api.cer
    3. Copy the saved thumbprint, edit the azure.properties file, find the certificateThumbprint property, and then paste the copied thumbprint value.
    4. Install the management certificate on the local machine’s certificate store, as follows:
      1. Click Start, click Run, type mmc, and then click OK.
      2. On the File menu, click Add/Remove Snap-in
      3. In the Available snap-ins pane, select Certificates, and then click Add
      4. In the Certificates snap-in dialog box, click Computer account, click Finish, and then in the Add or Remove Snap-ins dialog box click OK.
      5. In the navigation pane, expand the Certificates tree, right-click Personal, point to All Tasks, and then click Import.
      6. In the Certificate Import wizard, click Next, browse to the azure_api.pfx file, click Next, type the password for the file (123456), select Mark this key exportable, click Next, and then click Finish.

        Make sure when you browse for the file that the browse dialog box filter is PFX and not CER.

    5. Delete the PFX file.

      The certificate is saved in the certificate store and can be exported if required for a different computer.

  2. Upload the management certificate
    1. Login to your Azure Management Portal.
    2. On the views pane, click Hosted Services, Storage Accounts & CDN.
    3. On the navigation pane, click Management Certificates.
    4. Click Add Certificate.
    5. Browse to the azure_api.cer file, and then click OK.

Update the Blob Storage details

This section describes how to configure the blob storage that is used for storing the Cloudify distribution file and Cloudify images.

  1. Login to your Azure Management Portal.
  2. On the views pane, click Hosted Services, Storage Accounts & CDN.
  3. On the navigation pane, click Storage Accounts.

    If necessary, create a new storage account.

  4. Copy the Storage Account Name value, edit the azure.properties file, find the storageAccount property, and then paste the copied Storage Account Name value.
  5. Click View Access Key, copy the Primary Access Key value, edit the azure.properties file, find the storageAccessKey property, and then paste the copied View Access Key value.
