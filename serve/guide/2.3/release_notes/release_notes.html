Overview

Below is a list of new features, improvements, fixed issues, and known issues in Cloudify.
We track all the issues in our public JIRA.

Cloudify 2.3.1.1 GA – published on Feb 12, 2013
This version contains the following key improvements:

  • A fix for the logging.Levels JDK error – We fixed the logging.Levels error that occurs in jdk 1.7 U13 and 1.6 u39.
  • An OutOfMemory error fix – The OutOfMemory error when deploying large service files, has been fixed. More details about the fix can be found in our public JIRA.

The rest of this GA version is identical to Cloudify version 2.3.0 GA (see below).

Cloudify 2.3.0 GA – published on Dec 25, 2012
This version contains the following key improvement:

  • Finalization of Security – Cloudify introduces a new and exciting feature : security.

Cloudify 2.3.0 RC – published on Dec 09, 2012
This version contains the following key improvements:

  • Add templates- Cloudify now enables users to add cloud driver templates after bootstrapping.
  • Overriding cloud driver properties – Users can now override cloud driver’s properties during bootstrapping.

Cloudify 2.3.0 M2 – published on Nov 20, 2012
This version contains the following key improvements:

  • Multi Tenant Deployment – Cloudify now enables users to deploy several services on the same host.
  • BYON on Windows – Users can now install recipes on BYON on Windows machines as well.

Cloudify 2.3.0 M1 – published on Nov 8, 2012
This version contains the following key improvements:

  • DSL Overrides – Enables users to override application and service level properties at installation time, without changing the original recipe.
  • Disable Self Healing – An option to disable recipes self healing has been added. Now recipes can be activated in a ‘debug’ mode, in which there are no attempts to recover after failures. It can be achieved by setting the value of the -disableSelfHealing argument.
    This will allow recipes developers to debug and read log files after failures.
  • JClouds 1.5.2 – Cloudify now supports JClouds 1.5.2 GA for the default cloud driver.
