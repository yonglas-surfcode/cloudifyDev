Overview

Below is a list of new features, improvements, fixed issues, and known issues in Cloudify.
We track all the issues in our public JIRA.

Cloudify 2.5.1 GA – published on May 7, 2013
This version contains the following key improvements:

  • Debugger – Cloudify now enables users to debug Shell and Groovy scripts during runtime.
  • Recipe Repository – Cloudify enables users to install recipes from the Web UI.
  • Storage API – Users can now create, attach, detach or re-attach to a device from a recipe.

Cloudify 2.5.0 GA – published on Apr 10, 2013
This version contains the following key improvements:

  • Debugger – Cloudify now enables users to debug Shell and Groovy scripts during runtime.
  • Recipe Repository – Cloudify enables users to install recipes from the Web UI.
  • Storage API – Users can now create, attach, detach or re-attach to a device from a recipe.

Cloudify 2.5.0 RC – published on Mar 19, 2013
This version contains the following key improvements:

  • Configurable ports and memory – Cloudify now enables users to configure ports and memory usage.
  • Cloud driver structure change – Cloudify cloud driver file now has a new structure.

Cloudify 2.5.0 M7 – published on Feb 28, 2013
This version contains the following key improvements:

  • Configurable ports and memory – Cloudify now enables users to configure ports and memory usage.
  • Cloud driver structure change – Cloudify cloud driver file now has a new structure.

Cloudify 2.5.0 M5 – published on Feb 05, 2013
This version contains the following key improvements:

  • Multi Tenant Deployment – Cloudify now enables users to deploy several services on the management machines as well, via multitenancy.
  • Dynamic BYON support – VMs which cannot be used or accessed, are moved to the invalid VMs list. From now on, if there are no free machines, Cloudify tries to allocate machines from the invalid VMs list.
